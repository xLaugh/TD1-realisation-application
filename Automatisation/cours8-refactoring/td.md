# Maintenance Applicative
## Cours 8 - Refactoring
### Exercice 

#### 1 - Mise en place des tests
Avant de passer à la refactorisation, écrivez quelques tests pour vous assurer que tout fonctionne correctement.

#### 2 - Architecture
Pour commencer, essayez de retravailler l’architecture de l’application afin d’en améliorer la compréhension générale.  
Renommez également les fichiers pour plus de lisibilité et découpez-les différemment si nécessaire.  
Organisez le code de manière claire et cohérente.

#### 3 - Mise à jour des packages
Passez à Slim 4. Installez également les outils qui vous seront utiles pour la refactorisation.

#### 4 - Amélioration du code
Mettez le code aux derniers standards de PHP 8 (typage des fonctions, etc.).

#### 5 - Refactorisation
Essayez d’améliorer au maximum le code actuel sans modifier le fonctionnement de l’application : meilleur nommage des variables, factorisation, simplification des conditions, boucles, etc.

#### 6 - Ajout des logs
Ajoutez des logs pour chaque requête HTTP.

#### 7 - Ajout de la documentation
Utilisez un système de génération de documentation OpenAPI.

Si vous voyez d’autres points à ajouter, n’hésitez pas.
